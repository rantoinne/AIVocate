üîë Key Concepts:

  • AudioWorkletGlobalScope: Isolated JavaScript environment on audio thread - no access to DOM or main thread variables
  • AudioWorkletNode: Lives on main thread, acts as a proxy to communicate with the processor
  • Message Passing: The only way to communicate between main thread and audio thread
  • 128 Sample Chunks: Audio is processed in small, consistent chunks for low latency
  • Thread Isolation: Audio processing never blocks the UI, ensuring smooth performance
  • Same Output: Your WebSocket receives identical binary data as the old approach
